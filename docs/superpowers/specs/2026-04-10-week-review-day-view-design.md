**日期**：2026-04-10

## 背景

`weekReview.vue` 现有两种复制格式（文本 / JSON），通过一个滑动开关切换。本次新增第三种**展示模式**：按天视图，将本周有记录的任务按自然日聚合展示，并支持拖拽调整各任务当天工时，最终复制出符合周报要求的 JSON。

---

## 功能范围

### 视图切换

- 页头新增分段控件（Segmented Control），文案：**项目 / 按天**
- **项目视图**（默认）：保留现有卡片布局不变
- **按天视图**：新增，见下文

原有"文本 / JSON"滑动开关**移除**，改为页头下方一行说明文字：
- 项目视图下：`复制为文本格式`
- 按天视图下：`复制为 JSON 格式，拖拽调整各任务工时`

复制按钮行为自动跟随当前视图：项目视图复制文本，按天视图复制 JSON。

## 按天视图设计

### 数据聚合规则

按天视图采用**独立于现有 `doneItems`/`progressItems` 分组的新聚合逻辑**，直接遍历原始 `topics` 数据，按以下规则为每个 item 生成一条或多条「天-任务」条目：

| 条件 | 归天依据 | 显示类型 |
|------|---------|---------|
| `done = true` | `doneAt` 对应的自然日 | 已完成（✓） |
| `progress > 0 && !done` | `progressAt` 对应的自然日 | 进度中（…） |
| `done = true` 且 `progressAt` 存在且与 `doneAt` **不在同一自然日** | `progressAt` 对应的自然日**额外生成一条**进度中条目 | 进度中（…） |

**跨天 item 的多天展示（明确规则）**：若同一 item 的 `progressAt` 和 `doneAt` 落在不同自然日，则同时在两天出现：
- `progressAt` 那天：显示为「进度中」条目（使用该 item 当时的 `progress` 值）
- `doneAt` 那天：显示为「已完成」条目
- 两天的工时**完全独立**，互不影响，各自单独设置

边界条件：
- 仅显示**本周（周一 00:00 起）**内的条目，与现有页面的 `weekStartTs` 过滤一致
- `progress > 0 && !done` 但 `progressAt` 为 null 的 item **跳过**，不生成条目

结果按自然日从周一到周日排列，**跳过无任何条目的天**。每天内任务按 `doneAt`/`progressAt` 时间戳升序排列。

### 默认工时计算

- 每天默认总工时 **8 小时**
- 当天所有任务**均等分配**，按 **0.5h 向下取整**，最小 0.5h
- 计算公式：`Math.max(0.5, Math.floor((8 / count) * 2) / 2)`
- 余量（8h 减去所有任务工时之和）**直接舍弃**，不补给任何任务
- 例：3 个任务 → 各 2.5h（合计 7.5h，0.5h 余量舍弃）

### 拖拽交互

- 每个任务行右侧：进度条（拖拽把手在进度边界处）+ 小时数值
- 拖拽目标：进度条上的**圆形把手**，可左右拖动
- 步进 **0.5h**，范围 **0.5h ～ 24h**
- 进度条填充比例 = `hours / 24`（即 24h 时填满，8h 时填充 1/3）
- 拖动某任务时，其他任务工时**固定不变**（总工时可超过 8h）
- 调整为纯展示用途，**不持久化**，切换视图或刷新后恢复默认值

### 总工时提示

- 天标题右侧实时显示当天所有任务工时之和
- 总工时 ≤ 8h：灰色背景正常显示
- 总工时 > 8h：红色背景 + 红色数字提示（仅提示，不限制）

## 复制 JSON 逻辑（按天视图）

按天视图下点击"复制"，输出 JSON 数组。每个「天-任务」条目各生成**一条记录**：

```json
{
  "startDate": "YYYY.MM.DD",          // item 创建日期（item.id 时间戳转换）
  "lastUpdatedDate": "YYYY.MM.DD",    // 当天日期（即该条目归属的自然日）
  "hours": 4,                          // 用户拖拽设定的工时
  "cardId": null,
  "content": "主题标题 - 任务文本",    // 进度中条目追加（60%），已完成不追加
  "duration": [dayStartTs, dayEndTs],  // 当天 00:00:00.000 ～ 当天 23:59:59.999 的时间戳
  "plannedHours": 4                    // 同 hours
}
```

字段说明：
- `lastUpdatedDate`：固定为**当天日期**（该条目所属的自然日），与 item 的实际 `progressAt`/`doneAt` 无关
- `duration`：`[当天 00:00 时间戳, 当天 23:59:59.999 时间戳]`
- `hours` / `plannedHours`：用户拖拽值（默认为均分后的值）

与现有 JSON 格式的区别：
- `duration` 固定为**当天**的时间戳区间（不再是任务创建日～完成日）
- `hours` / `plannedHours` 由用户拖拽值决定（不再是 `days * 8`）
- 一个跨天任务会生成**多条**记录（每天一条）

## UI 结构变更摘要

```
页头
  ← 返回  本周完成  [项目 | 按天]  复制
  · 复制为文本格式                          ← 项目视图下
  · 复制为 JSON 格式，拖拽调整各任务工时    ← 按天视图下

按天视图内容区（可滚动）
  周一 4月7日 ────────────── 8h
    ✓  需求研究 · 登录页改版评审   [====◉  ] 4h
    …  API设计 · 权限模块接口     [====◉  ] 4h

  周三 4月9日 ────────────── 10h  ← 红色
    ✓  API设计 · 用户模块接口     [======◉] 6h  ← 红色
    …  API设计 · 权限模块接口     [====◉  ] 4h
```

## 不在范围内

- 工时数据持久化（不保存到 `topics.json`）
- 按天视图内的手动排序
- 跨天任务的工时联动（各天独立）
