## 背景

`weekReview.vue` 现有两种复制格式（文本 / JSON），通过一个滑动开关切换。本次新增第三种**展示模式**：按天视图，将本周有记录的任务按自然日聚合展示，并支持拖拽调整各任务当天工时，最终复制出符合周报要求的 JSON。

---

## 功能范围

### 视图切换

- 页头新增分段控件（Segmented Control），文案：**项目 / 按天**
- **项目视图**（默认）：保留现有卡片布局不变
- **按天视图**：新增，见下文

原有"文本 / JSON"滑动开关**移除**，改为页头下方一行说明文字：
- 项目视图下：`复制为文本格式`
- 按天视图下：`复制为 JSON 格式，拖拽调整各任务工时`

复制按钮行为自动跟随当前视图：项目视图复制文本，按天视图复制 JSON。

## 按天视图设计

### 数据聚合规则

遍历 `reviewTopics`（本周有记录的主题），将每个 item 按以下规则归天：

| item 类型 | 归天依据 |
|-----------|---------|
| 已完成（`done = true`） | `doneAt` 对应的自然日 |
| 进度中（`progress > 0, done = false`） | `progressAt` 对应的自然日 |

若一个 item 在多天有记录（如周一有 `progressAt`、周二完成），则在每天**各自独立出现**，工时单独设置。

结果按自然日从周一到周日排列，**跳过无记录的天**。

### 默认工时计算

- 每天默认总工时 **8 小时**
- 当天所有任务**均分** 8 小时（取整到 0.5h，最小 0.5h）
- 例：当天 3 个任务 → 各默认 2.5h（8 / 3 = 2.67 → 取 2.5）

### 拖拽交互

- 每个任务行右侧：小进度条 + 小时数值
- 左右拖动进度条调整当天工时，步进 **0.5h**，范围 **0.5h ～ 24h**
- 拖动某任务时，其他任务工时**固定不变**（总工时可超过 8h）
- 调整为纯展示用途，**不持久化**，刷新后恢复默认

### 总工时提示

- 天标题右侧实时显示当天所有任务工时之和
- 总工时 ≤ 8h：灰色背景正常显示
- 总工时 > 8h：红色背景 + 红色数字提示（仅提示，不限制）

## 复制 JSON 逻辑（按天视图）

按天视图下点击"复制"，输出 JSON 数组。每个 item 在每天各生成**一条记录**：

```json
{
  "startDate": "YYYY.MM.DD",       // item 创建日期（item.id）
  "lastUpdatedDate": "YYYY.MM.DD", // 当天日期
  "hours": 4,                      // 用户拖拽设定的工时
  "cardId": null,
  "content": "主题标题 - 任务文本", // 进度中追加（60%）
  "duration": [dayStartTs, dayEndTs], // 当天 00:00 ～ 当天 23:59:59.999 的时间戳
  "plannedHours": 4                // 同 hours
}
```

与现有 JSON 格式的区别：
- `duration` 固定为**当天**的时间戳区间（不再是任务创建日～完成日）
- `hours` / `plannedHours` 由用户拖拽值决定（不再是 `days * 8`）
- 一个跨天任务会生成**多条**记录（每天一条）

```
页头
  ← 返回  本周完成  [项目 | 按天]  复制

    …  API设计 · 权限模块接口     [====|  ] 4h

  周三 4月9日 ────────────── 10h ⚠
    ✓  API设计 · 用户模块接口     [======|] 6h  ← 红色
    …  API设计 · 权限模块接口     [====|  ] 4h
```
